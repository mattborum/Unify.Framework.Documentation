Using Enumerable Range

Namespace: Unify.Collections
Assembly: Unify.Collections.dll

The Enumerable Range feature allows UnifyEnumerators to iterate over a whole collection, part of a collection, repeat the iteration and step over items at a specified interval. This functionality is provided through the UnifyEnumerableRange class, which UnifyEnumerators expose through the enumerableRange property.

The UnifyEnumerableRange works with IUnifyObservableCollection or IUnifyObservableCollection<T> types to provide this functionality, it is not intended to be used in isolation.

Specifying a Range

An enumerable range is defined by a start item and an end item; these values are initialised by their index position using the startItemIndex and endItemIndex properties. It is the objects at these positions that Unify uses to maintain the enumerable range, not the index values themselves.

As the underlying collection changes through items being added, inserted or removed, the UnifyEnumerableRange will adjust so it is always defined by all items between, and including, the initial start and end items. If either of the start or end items are removed, the startItemIndex and endItemIndex will move to the next item inside the range.

When the startItemIndex and endItemIndex cannot find valid items to define the start and end of the range, the UnifyEnumerableRange becomes empty and cannot be used further.

To specify an enumerable range that always starts from the beginning of a collection, or always ends at the last item of a collection, regardless of any changes made; set startItemIndex and/or endItemIndex properties to -1. This is the default setting.

Looping a Range

Use the loopCount property to specify how many time an enumeration should be performed. This must be a value greater than or equal to 0, otherwise an ArgumentOutOfRangeException will be thrown.

Stepping over a Range

Items can be stepped over during an enumeration through the step property.

This must be a value greater than or equal to 1, otherwise an ArgumentOutOfRangeException will be thrown.
