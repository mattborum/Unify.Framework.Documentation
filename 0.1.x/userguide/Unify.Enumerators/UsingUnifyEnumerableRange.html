Using Unify Enumerable Range

Namespace: Unify.Collections, Unify.Collections.Generic
Assembly: Unify.Collections.dll

The behaviour of a Unify Enumerator is controlled through Unify's Enumerable Range feature, which is implemented by the UnifyEnumerableRange class. This class exposes the following properties:

Start Index

The startIndex property specifies the index from which the Unify Enumerator should start iterating over the underlying collection. This value is inclusive. The default value is specified by the UnifyEnumerableRange.AlwaysFirstItem constant; which is defined as always being the first item in the collection, regardless of any changes made.

End Index

The endIndex property specifies the index at which the Unify Enumerator should stop iterating over the underlying collction. This value is inclusive. The default value is specified by the UnifyEnumerableRange.AlwaysLastItem constant; which is defined as always being the last item in the collection, regardless of any changes made.

Loop Count

The loopCount property specifies how many times the Unify Enumerator should iterate over the underlying collection. The default value is 1, specified by the UnifyEnumerableRange.DefaultLoopCount constant.

Step

The step property specifies the interval at which the Unify Enumerator should move over the underlying collction, e.g. with a value of 1, Unify will iterate over every item in the collection between the startIndex and endIndex values; a value of 2 will iterate over every other item in the collection between the startIndex and endIndex values. The default value is 1, specified by the UnifyEnumerableRange.DefaultStep constant.

Locked

The locked property tells Unify Enumerators to only iterate over those items in the underlying collection that are between the startIndex and endIndex values inclusively, and covered by the step value, ignoring any items that are added to the collection after the Unify Enumerator has been created. Unify will not ignore items that are removed from the collection that are within the bounds of these values, those items will no longer be iterated over. The default value is false, specified by the UnifyEnumerableRange.DefaultLocked constant.

Creating a Unify Enumerable Range

A UnifyEnumerableRange instance can be created by using the new() operator. The constructor accepts the parameters; startIndex, endIndex, loopCount, step and locked. All parameters are optional, default values are used for any parameter not specified. A UnifyEnumerableRange instance is mutable.

Unify Enumerators provide two overloaded constructors to initialise Enumerable Range values; the first takes a UnifyEnumerableRange instance, and the other takes each of the Enumerable Range properties as optional parameters. The following code snippet shows how to create a Unify Enumerator with an Enumerable Range, using both constructor overloads:

IUnifyList<string> list = new UnifyList<string> { "Hello,", "Unify", "World!" };

// Create an enumerable range with unnamed parameter values for startIndex, endIndex, step, loopCount and locked
UnifyEnumerableRange range = new UnifyEnumerableRange(0, 4, 1, 2, false); 

// A more readable method using named parameters
UnifyEnumerableRange range = new UnifyEnumerableRange(startIndex: 0, endIndex: 3, step: 2, loopCount: 2, locked: true);

// Not all parameters have to be specified
UnifyEnumerableRange range = new UnifyEnumerableRange(endIndex: 4, loopCount: 2);

// Object Initializer Syntax
UnifyEnumerableRange range = new UnifyEnumerableRange() { startIndex = 8, endIndex = 1, step = 2 };

// Create an enumerator using the specified enumerable range
IUnifyEnumerator en = new UnifyEnumerator<string>(list, range);

// Create an enumerator using unnamed parameters
IUnifyEnumerator en = new UnifyEnumerator<string>(list, 3, 7, 2, 1, true);

// Use named parameters for readability
IUnifyEnumerator en = new UnifyEnumerator<string>(list, startIndex: 5, endIndex: 0, step: 2, loopCount: 2, locked: false);

// Not all parameters have to be specified
IUnifyEnumerator en = new UnifyEnumerator<string>(list, startIndex: 5, loopCount: 2, locked: true);

Although UnifyEnumerableRange instances are mutable, a Unify Enumerators Enumerable Range values are not; once a Unify Enumerator has Enumerable Range values specified, they cannot be changed from the Unify Enumerator instance.

Using startIndex and endIndex

This example shows each of the built-in Unify Enumerator types; UnifyEnumerator<T>, UnifyYoYoEnumerator<T> and UnifyRandomEnumerator<T>, each with a generic type T of string, to iterate over a .NET List<string> containing the words; "One", "Two", "Three", "Four" and "Five".

using System;
using Unify.Collections;
using Unify.Collections.Generic;
using UnityEngine;

public class Enumerators_UsingStartIndexAndEndIndex : MonoBehaviour
{
    private IUnifyList<string> myList;
    private IUnifyEnumerator<string> standardEnumerator;
    private IUnifyEnumerator<string> backwardsEnumerator;
    private IUnifyEnumerator<string> yoyoEnumerator;
    private IUnifyEnumerator<string> randomEnumerator;

    void Start()
    {
        myList = new UnifyList<string> { "One", "Two", "Three", "Four", "Five" };

        UnifyEnumerableRange range = new UnifyEnumerableRange(startIndex: 1, endIndex: 3);
        standardEnumerator = new UnifyEnumerator<string>(myList, range);
        backwardsEnumerator = new UnifyEnumerator<string>(myList, new UnifyEnumerableRange(startIndex: 3, endIndex: 1));
        yoyoEnumerator = new UnifyYoYoEnumerator<string>(myList, range);
        randomEnumerator = new UnifyRandomEnumerator<string>(myList, startIndex: 0, endIndex: 4);

        print("press S for STANDARD, B for BACKWARDS, Y for YO-YO, R for RANDOM, X to RESET ALL ENUMERATORS");
    }


    void Update()
    {
        if (Input.GetKeyUp(KeyCode.S))
        {
            print("--- STANDARD ENUMERATOR ---");
            while (standardEnumerator.MoveNext())
            {
                print(standardEnumerator.Current);
            }
            print(Environment.NewLine);
        }

        if (Input.GetKeyUp(KeyCode.B))
        {
            print("--- BACKWARDS ENUMERATOR ---");
            while (backwardsEnumerator.MoveNext())
            {
                print(backwardsEnumerator.Current);
            }
            print(Environment.NewLine);
        }

        if (Input.GetKeyUp(KeyCode.Y))
        {
            print("--- YO-YO ENUMERATOR ---");
            while (yoyoEnumerator.MoveNext())
            {
                print(yoyoEnumerator.Current);
            }
            print(Environment.NewLine);
        }

        if (Input.GetKeyUp(KeyCode.R))
        {
            print("--- RANDOM ENUMERATOR ---");
            while (randomEnumerator.MoveNext())
            {
                print(randomEnumerator.Current);
            }
            print(Environment.NewLine);
        }
    }
}

Exceptions

Exceptions will always be thrown under the following circumstances:

  • Setting the startIndex property to a value of less than 0 will throw an ArgumentOutOfRangeException
  • Setting the endIndex property to a value of less than 0 will throw an ArgumentOutOfRangeException
  • Setting the step property to a value of less than 1 will throw an ArgumentOutOfRangeException
  • Setting the loopCount property to a value of less than 1 will throw an ArgumentOutOfRangeException
