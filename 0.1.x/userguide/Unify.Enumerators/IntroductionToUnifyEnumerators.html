Introduction to Unify Enumerators

Namespace: Unify.Collections, Unify.Collections.Generic
Assembly: Unify.Collections.dll

This section provides examples to accompany the Unify API documentation on Unify Enumerators.

To get you up and running quickly, examples have been kept as simple as possible, using only the Unity Console.

Unify Enumerators allow the underlying collection to be modified while iterating; items can be added, inserted or removed during the iteration process, and Unify will adjust the enumerator on-the-fly. They also provide the ability to iterate over a whole collection, a partial collection, repeat an iteration and step over items at a specified interval. This functionality is provided by the Enumerable Range feature.

There are four types of Unify Enumerators; Forwards, Backwards, Yo-Yo and Random. Each of the Unify Enumerator types provide a generic and non-generic version, available through the following classes:

  • UnifyForwardsEnumerator<T> and UnifyForwardsEnumerator
  • UnifyBackwardsEnumerator<T> and UnifyBackwardsEnumerator
  • UnifyYoYoEnumerator<T> and UnifyYoYoEnumerator
  • UnifyRandomEnumerator<T> and UnifyRandomEnumerator

Unify Forwards Enumerator

Moves over a collection from the start of the enumerable range to the end of the enumerable range.

Unify Backwards Enumerator

Moves over a collection from the end of the enumerable range to the start of the enumerable range.

Unify Yo-Yo Enumerator

Moves over a collection from the start of the enumerable range to the end of the enumerable range and back to the start again.

Unify Random Enumerator

Moves over a collection randomly, within the bounds of the specified enumerable range, visiting each item once.

Events

Unify Enumerators provide the following events:

  • PassComplete - triggered each time a looping Unify Enumerator completes a pass of the loop
  • LoopFinished - triggered when a looping Unify Enumerator has completed all passes in the loop

Exceptions

Unify will always throw an exception when trying to set properties or parameters to an invalid value. Specific details are provided in each relevant section.
