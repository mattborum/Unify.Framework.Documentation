Introduction to Unify Enumerators

Namespace: Unify.Collections
Assembly: Unify.Collections.dll

This section provides examples to accompany the Unify API documentation on Unify Enumerators.

To get you up and running quickly, examples have been kept as simple as possible, using only the Unity Console.

Unify provides four enumerators; forwards, backwards, yo-yo and random.
Each enumerator has a generic and non-generic version, available through the following classes:

  • UnifyForwardsEnumerator<T> and UnifyForwardsEnumerator
  • UnifyBackwardsEnumerator<T> and UnifyBackwardsEnumerator
  • UnifyYoYoEnumerator<T> and UnifyYoYoEnumerator
  • UnifyRandomEnumerator<T> and UnifyRandomEnumerator

Unify Enumerators provide the ability to; iterate over just part of a collection, repeat the iteration and step over items at any specified interval.
This functionality is made possible by the Enumerable Range feature, which is provided by the UnifyEnumerableRange class that Unify Enumerators expose through the enumerableRange property.

Unify Forwards Enumerator

This Unify Enumerator iterates over a collection from the start index to the end index, as specified by the UnifyEnumerableRange.

Unify Backwards Enumerator

This Unify Enumerator iterates over a collection from the end index to the start index, as specified by the UnifyEnumerableRange.

Unify YoYo Enumerator

This Unify Enumerator iterates over a collection from the start index to the end index, and back to the start index again, as specified by the UnifyEnumerableRange.

Unify Random Enumerator

This Unify Enumerator iterates over a collection randomly, visiting each item once, within and including the start and end range values specified by the UnifyEnumerableRange.

Events

Unify Enumerators provide the following events:

  • PassComplete
  • LoopFinished

Exceptions

Unify does not throw operational exceptions on Unify Enumerators, but will always throw exceptions when properties are set to an invalid value.
The Unify Enumerator properties that currently throw exceptions are:

  • startItemIndex - Must be greater than or equal to -1
  • endItemIndex - Must be greater than or equal to -1
  • loopCount - Must be greater than or equal to 0
  • step - Must be greater than or equal to 1
