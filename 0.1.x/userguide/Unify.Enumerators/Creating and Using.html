Creating and Using Unify Enumerators

This section provides examples to accompany the Unify API documentation on Unify Collections.

The Unify.Collections namespace contains interfaces and classes to provide extra functionality to .NET collections.
The bulk of this functionality is made possible by Unify Enumerators, which allow you to iterate over a collection in many different ways.

To get you up and running quickly, examples have been kept as simple as possible, using only the Unity Console.

Unify Enumerators

Unify provides the following enumerators out of the box; forwards, backwards, yoyo and random, or, if you need something different you can write your own custom enumerator or request one.

Unify Enumerators are stand-alone. You can create an instance of a Unify Enumerator to operate on any Unify/.NET collection, including collections which .NET enumerators do not currently support, such as jagged arrays.

Unify Enumerators can be reused, there is no need to create a new() enumerator or use GetEnumerator(), simply call the Unify Emumerator's Reset() method and use again.

Enumerable Range

Unify Enumerators provide an Enumerable Range feature, this allows you to; iterate over just a portion of the underlying collection, repeat the iteration any number of times and step over items at any interval. The Enumerable Range is dynamic, allowing values to be changed at any time, even while iterating over a collection, and Unify will automatically grow and shrink the Enumerable Range as these values change or items are added, inserted and removed from the collection.

Unify Collections

All Unify Collections implement Unify Enumerators.

Unify Collections are observable through the CollectionChanged event. The CollectionChanged event is used to notify subscribers when items are added, inserted or removed, or when a collecton is cleared or reset.

Unify Collections offer generic and non-generic versions of a collection.

The Unify.Collections namespace currently offers two collections taking advantage of this extra functionality, the UnifyList and the generic UnifyList<T>.
