Looping Unify Enumerators

Namespace: Unify.Collections, Unify.Collections.Generic
Assembly: Unify.Collections.dll

The UnifyEnumerableRange class provides a loopCount property to specify how many times a Unify Enumerator should repeat an iteration. The default setting for loopCount is 1.

Events

Unify Enumerators provide the following events:

  • PassComplete - triggered each time a looping Unify Enumerator successfully completes a pass of the loop
  • LoopFinished - triggered when a looping Unify Enumerator successfully completes all passes in a loop

Example

This example demonstrates each of the Unify Enumerator types, with a generic type T of string, to iterate over a list containing the colours of the rainbow; "Red", "Orange", "Yellow", "Green", "Blue", "Indigo" and "Violet". The loopCount property is set to 2 to repeat the enumeration twice. To demonstrate when Unify Enumerator events are triggered, the PassComplete and LoopFinished events are used to print a message to the Unity Console.

Attach the script below to a GameObject and press:

  • F to Start the Forwards Enumerator
  • B to Start the Backwards Enumerator
  • Y to Start the Yo-Yo Enumerator
  • R to Start the Random Enumerator
  • X to Reset all Enumerators
using System.Collections.Generic;
using Unify.Collections.Generic;
using UnityEngine;

public class Enumerators_LoopingUnifyEnumerators : MonoBehaviour
{
    private IList<string> list;
    private IUnifyEnumerator<string> forwards;
    private IUnifyEnumerator<string> backwards;
    private IUnifyEnumerator<string> yoyo;
    private IUnifyEnumerator<string> random;

    void Awake()
    {
        list = new List<string> { "Red", "Orange", "Yellow", "Green", "Blue", "Indigo", "Violet" };

        forwards = new UnifyForwardsEnumerator<string>(list);
        forwards.enumerableRange.loopCount = 2;
        forwards.PassComplete += (s, e) => { print("*** PASS COMPLETE ***"); };
        forwards.LoopFinished += (s, e) => { print("*** LOOP FINISHED ***"); };

        backwards = new UnifyBackwardsEnumerator<string>(list);
        backwards.enumerableRange.loopCount = 2;
        backwards.PassComplete += (s, e) => { print("*** PASS COMPLETE ***"); };
        backwards.LoopFinished += (s, e) => { print("*** LOOP FINISHED ***"); };

        yoyo = new UnifyYoYoEnumerator<string>(list);
        yoyo.enumerableRange.loopCount = 2;
        yoyo.PassComplete += (s, e) => { print("*** PASS COMPLETE ***"); };
        yoyo.LoopFinished += (s, e) => { print("*** LOOP FINISHED ***"); };

        random = new UnifyRandomEnumerator<string>(list);
        random.enumerableRange.loopCount = 2;
        random.PassComplete += (s, e) => { print("*** PASS COMPLETE ***"); };
        random.LoopFinished += (s, e) => { print("*** LOOP FINISHED ***"); };

        print("press F for FORWARDS, B for BACKWARDS, Y for YO-YO, R for RANDOM, X to RESET ALL ENUMERATORS");
    }

    void Update()
    {
        if (Input.GetKeyUp(KeyCode.F))
        {
            print("--- Forwards Enumerator ---");
            while (forwards.MoveNext())
            {
                print(forwards.Current);
            }
        }

        if (Input.GetKeyUp(KeyCode.B))
        {
            print("--- Backwards Enumerator ---");
            while (backwards.MoveNext())
            {
                print(backwards.Current);
            }
        }

        if (Input.GetKeyUp(KeyCode.Y))
        {
            print("--- Yo-Yo Enumerator ---");
            while (yoyo.MoveNext())
            {
                print(yoyo.Current);
            }
        }

        if (Input.GetKeyUp(KeyCode.R))
        {
            print("--- Random Enumerator ---");
            while (random.MoveNext())
            {
                print(random.Current);
            }
        }

        if (Input.GetKeyUp(KeyCode.X))
        {
            print("--- Resetting Enumerators ---");
            forwards.Reset();
            backwards.Reset();
            yoyo.Reset();
            random.Reset();
        }
    }
}

Exceptions

Exceptions will always be thrown under the following circumstances:

  • Setting the loopCount property to a value of less than 0 will throw an ArgumentOutOfRangeException
