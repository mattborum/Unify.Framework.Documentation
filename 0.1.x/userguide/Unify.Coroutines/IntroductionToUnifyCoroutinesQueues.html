Introduction to Unify Coroutine Queues

Namespace: Unify.Coroutines
Assembly: Unify.Coroutines.dll

The UnifyCoroutineQueue is a specialised container created to execute a group of Unify Coroutines in sequence. When the Unify Coroutine Queue is started the first Unify Coroutine in the queue will be executed, only when that has finished will the next Unify Coroutine be started.

This collection accepts any class derived from UnifyCoroutineBase, this includes generic and non-generic Unify Coroutines. The UnifyCoroutineCollection class also derives from UnifyCoroutineBase, making it also a Unify Coroutine. This means it has all the functionality of a UnifyCoroutine; it can be paused, resumed, cancelled and reused; the difference being operations work on all Unify Coroutines in the queue.

As a Unify Coroutine Queue is also a Unify Coroutine, this means a Unify Coroutine Queue can be added to another Unify Coroutine Queue or a Unify Coroutine Collection.
