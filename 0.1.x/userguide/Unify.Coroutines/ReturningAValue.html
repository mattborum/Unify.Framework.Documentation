Returning a value from a Unify Coroutine

Namespace: Unify.Coroutines
Assembly: Unify.Coroutines.dll

To return a value from a Unify Coroutine, you should use the generic UnifyCoroutine<T> class. The generic type parameter <T> represents the type of the value to be returned.
To create a UnifyCoroutine<T> instance, use the new() operator, specify the return type using <T> and pass the Coroutine method to the constructor.

The return type of the Unify Coroutine should always be IEnumerator, regardless of the type being returned.
Use yield return to return the value, rather that just return.

For example, to create a UnifyCoroutine<T> instance that returns a string:

UnifyCoroutine<string> myUnifyCoroutine = new UnifyCoroutine<string>(SayHello());

IEnumerator SayHello()
{
    yield return  "Hello, Unify!";
}

Capturing the return value

The UnifyCoroutine<T> class provides a ValueReturned event for capturing return values.
The ValueReturned event is fired every time a value of type <T> is returned from a yield return statement.

An instance of UnifyCoroutineEventArgs<T> is passed to the event handler, the return value is held in the returnValue property, which is of type <T>.

See: How to subscribe to and unsubscribe from Unify Coroutine events.

Example One

This Unify Coroutine example has a startDelay of 2 seconds and then returns a string with the value "Hello, Unify!". It uses a UnifyCoroutine<string> instance to return a value of type string and a lambda expression as the ValueReturned event handler to capture the return value and print it to the Unity Console. Here, failSilently has been set to false to allow you to trigger exceptions as described below.

Attach the script below to a GameObject and press:

  • S to Start the Unify Coroutine
using System.Collections;
using Unify.Coroutines;
using UnityEngine;

public class Coroutines_ReturnValueOnce : MonoBehaviour
{
    private UnifyCoroutine<string> myUnifyCoroutine;

    void Start()
    {
        myUnifyCoroutine = new UnifyCoroutine<string>(WaitAndReturnString(), startDelay: 2f) { failSilently = false, keepAlive = true };

        myUnifyCoroutine.ValueReturned += (sender, e) =>
        {
            print(e.returnValue);
        };

        print("press S to START");
    }

    void Update()
    {
        if (Input.GetKeyUp(KeyCode.S))
        {
            print("* Starting Unify Coroutine - waiting for 2 seconds....");
            myUnifyCoroutine.Start();
        }
    }

    IEnumerator WaitAndReturnString()
    {
        yield return new WaitForSecondsRealtime(2f);
        yield return "Hello, Unify!";
    }
}

Example Two

This Unify Coroutine example counts from 1 to 30 and, at 1 second intervals, returns an integer value every time the counter is a multiple of 3.
It uses a UnifyCoroutine<int> instance to return a value of type integer and a programmatical event handler to capture the return value and print it to the Unity Console.
Here, failSilently has been set to false to allow you to trigger exceptions as described below.

Attach the script below to a GameObject and press:

  • S to Start the Unify Coroutine
using System.Collections;
using Unify.Coroutines;
using UnityEngine;

public class Coroutines_ReturnValueMultiple : MonoBehaviour
{
    private UnifyCoroutine<int> myUnifyCoroutine;

    void Start()
    {
        myUnifyCoroutine = new UnifyCoroutine<int>(MultiplesOfThree());
        myUnifyCoroutine.failSilently = false;
        myUnifyCoroutine.keepAlive = true;

        myUnifyCoroutine.ValueReturned += ValueReturnedHandler;

        print("press S to START");
    }

    void Update()
    {
        if (Input.GetKeyUp(KeyCode.S))
        {
            print("* Starting Unify Coroutine *");
            myUnifyCoroutine.Start();
        }
    }

    private void ValueReturnedHandler(object sender, UnifyCoroutineEventArgs<int> e)
    {
        print($"{e.returnValue} is a multiple of 3");
    }

    IEnumerator MultiplesOfThree()
    {
        for (int i = 1; i <= 30; i++)
        {
            if (i % 3 == 0)
            {
                yield return i;
                yield return new WaitForSecondsRealtime(1f);
            }
        }
    }
}

Exceptions

Exceptions will always be thrown under the following circumstances:

  • Passing a null value to the target parameter

Unify does not throw exceptions specifically around returning a value from a Unify Coroutine, however exceptions will be thrown from other operations:

  • Creating and Starting
  • Pausing and Resuming
  • Stopping and Restarting
  • Cancelling
