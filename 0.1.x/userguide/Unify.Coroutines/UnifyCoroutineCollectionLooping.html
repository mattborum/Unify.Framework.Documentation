Looping a Unify Coroutine Collection

Namespace: Unify.Coroutines
Assembly: Unify.Coroutines.dll

Looping a Unify Coroutine Collection is achieved through the loopCount property, the same way a Unify Coroutine is looped; the loopCount property specifies how many times the Unify Coroutine Collection should be repeated. One pass of a Unify Coroutine Collection loop is considered completed when all Unify Coroutines in the collection are finished. Only Unify Coroutines in the collection whose keepAlive property has a value of true will be repeated.

The passDelay property is used to delay the starting of the next pass in the loop.

Example One

This example adds three Unify Coroutines to a Unify Coroutine Collection; "PrintDaysOfWeek()", "PrintColoursOfTheRainbow()" and "SayHello()". Each Coroutine runs for a different length of time to demonstrate that the PassComplete and Finished events are triggered only when all Unify Coroutines in the Unify Coroutine Collection have finished. The keepAlive property of the "SayHello()" Unify Coroutine is set to false to show how to prevent Unify Coroutines in a Unify Coroutine Collection from looping.

Attach the script below to a GameObject and press:

  • S to Start the Unify Coroutine Collection
using System.Collections;
using Unify.Coroutines;
using UnityEngine;

public class Coroutines_Collection_Looping_One : MonoBehaviour
{
    private UnifyCoroutineCollection collection;

    void Awake()
    {
        collection = new UnifyCoroutineCollection { loopCount = 3, name = "Martyn" };

        collection.PassComplete += (sender, args) => { print("--- Pass Complete ---"); };
        collection.Finished += (sender, args) => { print("--- Finished ---"); }; 

        collection.Add(new UnifyCoroutine(PrintDaysOfWeek()) { keepAlive = true } );
        collection.Add(new UnifyCoroutine(PrintColoursOfTheRainbow()) { keepAlive = true });
        collection.Add(new UnifyCoroutine(SayHello()) { keepAlive = false });

        print("press S to START");
    }

    void Update()
    {
        if (Input.GetKeyUp(KeyCode.S))
        {
            collection.Start();
        }
    }

    private IEnumerator PrintDaysOfWeek()
    {
        string[] days = { "Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday", "Sunday" };
        foreach (string day in days)
        {
            print(day);
            yield return new WaitForSecondsRealtime(0.4f);
        }
    }

    private IEnumerator PrintColoursOfTheRainbow()
    {
        string[] colours = { "Red", "Orange", "Yellow", "Green", "Blue", "Indigo", "Violet" };
        foreach (string colour in colours)
        {
            print(colour);
            yield return new WaitForSecondsRealtime(1f);
        }
    }

    private IEnumerator SayHello()
    {
        for (int i = 0; i < 10; i++)
        {
            print("Hello, World!");
            yield return new WaitForSecondsRealtime(0.9f);
        }
    }
}

Example Two

This example shows how a looping Unify Coroutine Collection behaves when it contains a looping Unify Coroutine. Two Unify Coroutines, CountToThree() and SayHello(), are added the collection. CountToThree() has a loopCount of 3 and a passDelay of 2.5 seconds; it also uses the PassComplete event to print a message to the Unity Console at the end of each pass. SayHello() prints a greeting to the Unity Console in 5 languages; English, French, Spanish, German and Italian. The loopCount property of SayHello() is not set, so the default value of 1 will be used.

The Unify Coroutine Collection has a loopCount value of 2 and passDelay value of 5 seconds; it's PassComplete and Finished events are used to print a message to the Unity Console to demonstrate how a Unify Coroutine Collection waits for all Unify Coroutines to finish before considering a pass of the loop completed.

In this scenario, the Unify Coroutine Collection will consider the 3 loops of CountToThree() as one pass of it's own loopCount value of 2. SayHello() will be kept waiting until CountToThree() has completed one pass, which is 3 loops.

Attach the script below to a GameObject and press:

  • S to Start the Unify Coroutine Collection
using System.Collections;
using Unify.Coroutines;
using UnityEngine;

public class Coroutines_Collection_Looping_Two : MonoBehaviour
{
    private UnifyCoroutineCollection collection;

    void Awake()
    {
        collection = new UnifyCoroutineCollection(loopCount: 2, passDelay: 5f)
        {
            new UnifyCoroutine(CountToThree(), loopCount: 3, passDelay: 2.5f, keepAlive: true,
                passCompleteEventHandler: (s, e) =>
                {
                    print("--- CountToThree() Pass Complete ---");
                }),
            new UnifyCoroutine(SayHello()) { keepAlive = true }
        };

        collection.PassComplete += (sender, args) => { print("--- Collection PassComplete ---"); };
        collection.Finished += (sender, args) => { print("--- Collection Finished ---"); };

        print("press S to START");
    }


    void Update()
    {
        if (Input.GetKeyUp(KeyCode.S))
        {
            collection.Start();
        }
    }

    private IEnumerator SayHello()
    {
        string[] s = { "Hello", "Bonjour", "Hola", "Hallo", "Buongiorno" };
        for (int i = 0; i < s.Length; i++)
        {
            print(s[i]);
            yield return new WaitForSecondsRealtime(1.2f);
        }
    }

    private IEnumerator CountToThree()
    {
        for (int i = 1; i <= 3; i++)
        {
            print(i);
            yield return new WaitForSecondsRealtime(0.8f);
        }
    }
}

Exceptions

Exceptions will always be thrown under the following circumstances:

  • Passing a null value to the target parameter
  • Passing a value of less than 0 to the loopCount property or parameter
  • Passing a value of less than 0 to the passDelay property or parameter
