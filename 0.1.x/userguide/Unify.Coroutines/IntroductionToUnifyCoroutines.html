Introduction to Unify Coroutines

Unify extends the core functionality of Unity's Coroutines.

Unify Coroutines add the ability to:

  • Delay Starting Coroutines
  • Pause Coroutines
  • Resume Coroutines
  • Cancel Coroutines on a Per Instance Basis
  • Restart/Reuse Coroutines (without having to create a new() instance)
  • Return a Value from a Coroutine
  • Loop a Coroutine

Unify Coroutines also provide the following events:

  • Started
  • Paused
  • Resumed
  • Stopped
  • Cancelled
  • ValueReturned
  • LoopComplete
  • Finished

What's the difference between Pause, Stop and Cancel?

Pause()

Temporarily halts execution until the Resume() method is called. Execution resumes from the point is was paused.

Stop()

Halts execution. The Unify Coroutine can be restarted with the Start() method. Execution starts from the beginning.

Cancel()

Permanently halts execution. The Unify Coroutine cannot be resumed or restarted.

Known Issues

When a Unify Coroutine is executing a CustomYieldInstruction, such as WaitForSeconds() , this does not currently pause the CustomYieldInstruction.
