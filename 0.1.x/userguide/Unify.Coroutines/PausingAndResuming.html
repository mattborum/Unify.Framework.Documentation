Pausing and Resuming a Unify Coroutine

A Unify Coroutine can be paused and resumed.

To pause a Unify Coroutine, call the Pause() method on a Unify Coroutine instance. The Pause() method will return true if the Unify Coroutine was paused successfully, otherwise it will return false.

bool paused = myUnifyCoroutine.Pause();

To resume a Unify Coroutine, call the Resume() method on a Unify Coroutine instance. The Resume() method will return true if the Unify Coroutine was resumed successfully, otherwise it will return false.

bool resumed = myUnifyCoroutine.Resume();

Example

This Unify Coroutine example runs for 60 seconds, printing the current second value to the Unity Console at 1 second intervals.
Here, failSilently has been set to false to allow you to trigger exceptions as described below. The name property has been set to 'Karen' to demonstrate how Unify Coroutine instances are identified in exception messages.

Attach the script below to a GameObject and press:

  • S to Start
  • P to Pause
  • R to Resume
using System.Collections;
using Unify.Coroutines;
using UnityEngine;

public class PausingAndResuming : MonoBehaviour
{
    private UnifyCoroutine myUnifyCoroutine;

    void Start()
    {
        myUnifyCoroutine = new UnifyCoroutine(WaitAndPrint(), name: "Karen") { failSilently = false };

        print("press S to START, P to PAUSE, R to RESUME");
    }

    void Update()
    {
        if (Input.GetKeyUp(KeyCode.S))
        {
            print("* Starting Unify Coroutine *");
            myUnifyCoroutine.Start();
        }

        if (Input.GetKeyUp(KeyCode.P))
        {
            print("* Pausing Unify Coroutine *");
            myUnifyCoroutine.Pause();
        }

        if (Input.GetKeyUp(KeyCode.R))
        {
            print("* Resuming Unify Coroutine *");
            myUnifyCoroutine.Resume();
        }
    }

    IEnumerator WaitAndPrint()
    {
        for (int i = 0; i < 60; i++)
        {
            print("i = " + i);
            yield return new WaitForSecondsRealtime(1f);
        }
    }
}

Exceptions

Exceptions will always be thrown under the following circumstances:

  • Passing a null value to the target parameter

Exceptions will be thrown under the following circumstances, only if failSilently is false:

  • Pausing a Unify Coroutine that is in a state other than Running
  • Resuming a Unify Coroutine that is in a state other than Paused
