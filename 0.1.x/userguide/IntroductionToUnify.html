Introduction to Unify

This section provides examples to accompany the Unify API documentation.

To get you up and running quickly, examples have been kept as simple as possible, using only the Unity Console.

This 0.1.x release of Unify features the Unify Coroutine portion of the framework.

Future releases of Unify will see the following features added;

Easing Functions

Unify provides many, many easing functions. You can use the lightweight static easing functions, or if you need something more powerful, Unify provides a feature rich Coroutine version of each easing function. The Unify Coroutine version of the easing functions offer all the Unify Coroutine features; pausing, resuming, looping and cloning/reuse to name a few, plus extra functionality including a progress update event and the choice of an absolute or relative end value. You can even bind a GameObject's transform to an easing function to manipulate a GameObject without any extra code. Unify Easing Functions currently support the following types; float, Vector2, Vector3, Vector4 and Color.

Unify Enumerators

Iterate over any .NET collection any way you please; forwards, backwards, yoyo, random, or write your own custom enumerator *. Iterate over just a portion of a collection using Unify's range features, which also offer stepping. Combine range and stepping with any Unify Enumerator. You can choose to have the range automatically grow and shrink as items are added and removed from the collection. Unify Enumerators can be cloned for reuse, no need to create a new enumerator every time.

Unify Collections

Unify Collections allow you to add and remove items during a loop. Unify Collections are observable, subscribe to events when items are added, inserted or removed from a Unify Collection, or the collecton is cleared. Unify provides generic and non-generic versions. Use all the features of Unify Enumerators with Unify Collections.

* Available with the source code release of Unify.
