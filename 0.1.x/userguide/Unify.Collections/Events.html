Unify Collection Events

Namespace: Unify.Collections, Unify.Collections.Generic
Assembly: Unify.Collections.dll

Unify Collections provide the following events:

  • CollectionChanged and CollectionChanged<T> - triggered when an item is added, inserted or removed
  • CollectionCleared - triggered when either a generic Unify Collection's Clear() method, or non-generic Unify Collection's Clear() method is called

This functionality is exposed through the IUnifyObservableCollection and IUnifyObservableCollection<T> interface types, which all Unify Collections implement.

Subscribing to Unify Collection events

There are three ways to subscribe to Unify Collection events; programmatically, anonymous methods and lambda expressions:

Programmatically

myUnifyCollection.CollectionChanged += CollectionChangedEventHandler;

private void CollectionChangedEventHandler(object sender, UnifyCollectionChangedEventArgs e)
{
	print("* UNIFY COLLECTION CHANGED EVENT HANDLER *");
}

Anonymous Method

myUnifyCollection.CollectionChanged += delegate(object sender, UnifyCollectionChangedEventArgs e)
{
	print("* UNIFY COLLECTION CHANGED EVENT HANDLER *");
};

Lambda Expression

myUnifyCollection.CollectionChanged += (sender, e) =>
{
	print("* UNIFY COLLECTION CHANGED EVENT HANDLER *");
};

If a Unify Collection event is subscribed to via either an anonymous method or lambda expression, then it is not possible to unsubscribe from that event unless using a reference to the original method which was used to subscribe:

Anonymous Method

private EventHandler<UnifyCollectionChangedEventArgs> myUnifyCollectionChangedEventDelegate;
myUnifyCollectionChangedEventDelegate = delegate { print("* UNIFY COLLECTION CHANGED EVENT HANDLER *"); };

myUnifyCollection.CollectionChanged += myUnifyCollectionChangedEventDelegate;  // Subscribe
myUnifyCollection.CollectionChanged -= myUnifyCollectionChangedEventDelegate;  // Unsubscribe

Lambda Expression

private EventHandler<UnifyCollectionChangedEventArgs> myUnifyCollectionChangedEventDelegate;
myUnifyCollectionChangedEventDelegate = (sender, args) => { print("* UNIFY COLLECTION CHANGED EVENT HANDLER *"); };

myUnifyCollection.CollectionChanged += myUnifyCollectionChangedEventDelegate;  // Subscribe
myUnifyCollection.CollectionChanged -= myUnifyCollectionChangedEventDelegate;  // Unsubscribe

EventArgs

Unify Collections provide generic UnifyCollectionChangedEventArgs<T> and non-generic UnifyCollectionChangedEventArgs classes, an instance of which is passed to event handlers. These classes provide information about the change to the collection; the action property is a UnifyCollectionChangedAction type, that specifies the operation causing the change; the item property, of type T for generic collections and of type object for non-generic collections, contains the item responsible for the change and the index property is the index position of that item.

Example

This example uses a UnifyList<T>, with a generic type T of char, initialised with the values; 'A', 'B', 'C', 'D', 'E', to demonstrate the CollectionChanged and CollectionCleared events. Items can be added, inserted or removed from the collection causing the CollectionChanged event to be triggered, and the collection can be cleared to trigger the CollectionCleared event.

Attach the script below to a GameObject and press:

  • A to Add an item to end of the list
  • C to Clear the list
  • I to Insert an item into the list at a random position
  • R to Remove an item from the list at a random position
  • P to Print the list
using System;
using System.Text;
using Random = System.Random;
using Unify.Collections.Generic;
using UnityEngine;

public class Collections_Events : MonoBehaviour
{
    private IUnifyList<char> list;
    private Random rnd;

    void Awake()
    {
        rnd = new Random();
        list = new UnifyList<char> { 'A', 'B', 'C', 'D', 'E' };

        list.CollectionChanged += list_CollectionChanged;
        list.CollectionCleared += (sender, args) =>
        {
            print("--- Collection Cleared ---");
            PrintList();
            print(Environment.NewLine);
        };

        print("press A to Add(), C to Clear(), R to RemoveAt(), I to Insert(), P to Print List");
        PrintList();
    }

    void Update()
    {
        if (Input.GetKeyUp(KeyCode.A))
        {
            char c = Convert.ToChar(rnd.Next(65, 91));
            list.Add(c);
        }

        if (Input.GetKeyUp(KeyCode.C))
        {
            list.Clear();
        }

        if (Input.GetKeyUp(KeyCode.I))
        {
            int i = rnd.Next(0, list.Count);
            char c = Convert.ToChar(rnd.Next(65, 91));
            list.Insert(i, c);
        }

        if (Input.GetKeyUp(KeyCode.R))
        {
            int r = rnd.Next(0, list.Count);
            list.RemoveAt(r);
        }

        if (Input.GetKeyUp(KeyCode.P))
        {
            PrintList();
        }
    }


    private void list_CollectionChanged(object sender, UnifyCollectionChangedEventArgs<char> e)
    {
        print("--- Collection Changed Event ---");
        print($"Action: {e.action},  Item: {e.item},  Index: {e.index}");
        PrintList();
        print(Environment.NewLine);
    }

    private void PrintList()
    {
        StringBuilder sb = new StringBuilder("Current List: ");

        for (int i = 0; i < list.Count; i++)
        {
            sb.Append(list[i]);

            if (i < list.Count - 1)
            {
                sb.Append(", ");
            }
        }

        print(sb);
    }
}

Exceptions

Exceptions will always be thrown under the following circumstances:

  • Removing an item from an empty collection will throw an ArgumentOutOfRangeException
