Introduction to Unify

This is version 0.1.0 of Unify. This release features the Unify Coroutine portion of the framework.

Unify Coroutines

Unify Coroutines extend Unity Coroutines to provide much requested extra functionality. Unify Coroutines add the following features; delayed start, pause, resume, cancel, restart/reuse, return a value and looping. Unify Coroutines are observable, subscribe to events to notified when a Unify Coroutine is; started, paused, resumed, stopped, cancelled or finished as well as events to capture a return value and callback after every pass of a looping Unify Coroutine.

See the Unify Coroutine User Guide for more details.

Unify Easing Functions

Unify provides many easing functions. You can use the lightweight static easing functions provided, or if you need something more powerful, Unify also provides a feature rich Coroutine version of each easing function. The Unify Coroutine version of easing functions offer all the Unify Coroutine features; pausing, resuming, looping and cloning/reuse to name a few, plus extra functionality including a progress update event and the choice of an absolute or relative end value. You can even bind a GameObject's transform to an easing function to manipulate a GameObject without any extra code. Unify's easing functions currently support easing of the following types; float, Vector2, Vector3, Vector4, Quaternion and Color.

Unify Enumerators

Iterate over any .NET collection any way you please. Unify provides the following enumerators out of the box; forwards, backwards, yoyo and random. Or if you need something different, you can write your own custom enumerators*. Move over just a portion of a collection using Unify's range feature, which also offers stepping. Combine range and stepping with any Unify Enumerator. You can choose to have the range automatically grow and shrink as items are added, inserted and removed. Unify Enumerators can be cloned for reuse, no need to create a new enumerator every time, just call the Reset() method and use again.

Unify Collections

Unify Collections allow you to iterate over a collection while adding, inserting and removing items at the same time. Unify Collections are observable, subscribe to events to be notified when items are added, inserted or removed, or the collecton is cleared. Unify provides a generic and non-generic version of the Collection. Use all the features of Unify Enumerators with Unify Collections.

The Unify.Collection namespace also offers two specialised collections for Unify Coroutines; the UnifyCoroutineQueue and the UnifyCoroutineCollection. These two collections operate in the same way as a standard .NET Queue or Collection, but have been built to allow fine grained control of Unify Coroutines.

With a Unify Coroutine Queue you can line up a chain of Unify Coroutines to be executed one after the other, or alternatively, the Unify Coroutine Collection allows you to start a group of Unify Coroutines at the same time.

In fact, the Unify Coroutine Queue and Unify Coroutine Collection classes are also Unify Coroutines themselves. You can control a whole queue or collection of Unify Coroutines the same way you can a single Unify Coroutine. You have all the functionality of pausing, restaring and reusing the whole collection just like a single Unify Coroutine. This means they are also interchangable; you could, for example, add a Unify Coroutine Collection to a Unify Coroutine Queue or vice versa.

As Unify Easing Functions are also Unify Coroutines, these can be added to a Unify Queue or Collection and controlled in the same way.
