Introduction to Unify

The Unify Framework is a collection of libraries aimed at making Unity development a much quicker and smoother experience. The current plan for Unify is to improve Unity Coroutines and add an easing library which takes advantage of the new Coroutine functionality, along with improvements to .NET enumerators and collections.

This is version 0.1.0 of Unify. This release features the Unify Coroutine portion of the framework.

Unify Coroutines

Unify Coroutines extend Unity Coroutines to provide much requested extra functionality. Unify Coroutines add the following features; delayed start, pause, resume, cancel, restart/reuse, return a value and looping. Unify Coroutines are observable, subscribe to events to be notified when a Unify Coroutine is; started, paused, resumed, stopped, cancelled or finished as well as events to capture a return value and callback after every pass of a looping Unify Coroutine.

See the Unify Coroutine User Guide for more details.

Unify Roadmap

Unify 0.2.0 (Q1 2021)

Unify Enumerators

Iterate over any .NET collection any way you please. Unify provides the following enumerators out of the box; forwards, backwards, yoyo and random, or if you need something different, you can write your own custom enumerators*. Operate on just a portion of a collection using Unify's range feature, which also offers stepping. Combine range and stepping with any Unify Enumerator. You can choose to have the range automatically grow and shrink as items are added, inserted and removed.

Unify Enumerators can be cloned for reuse, no need to create a new enumerator every time.

Unify Enumerators are stand-alone. You can create instances of Unify Enumerators to operate on any Unify/.NET collection, including collections which .NET does not currently support, such as jagged arrays.

Unify Collections

Unify Collections are observable, subscribe to events to be notified when items are added, inserted or removed, or when a collecton is cleared or reset.

Use all the features of Unify Enumerators with Unify Collections.

Unify Collections allow you to iterate over a collection at the same time as adding, inserting or removing items.

Unify also offers two specialised collections for Unify Coroutines; the UnifyCoroutineQueue and the UnifyCoroutineCollection. These two collections operate in the same way as a standard .NET Queue or Collection, but have been built to allow fine grained control of Unify Coroutines. With a Unify Coroutine Queue you can line up a chain of Unify Coroutines to be executed one after the other, or alternatively, the Unify Coroutine Collection allows you to start a group of Unify Coroutines at the same time.

The Unify Coroutine Queue and Unify Coroutine Collection classes are also Unify Coroutines themselves. You can control a whole queue or collection of Unify Coroutines the same way you control a single Unify Coroutine and still have all the functionality of; delayed start, pause, resume, cancel, restart/reuse, returning a value and looping.

The Unify Coroutine Queue and Unify Coroutine Collection classes are built on top of Unify Coroutines, this means they are also interchangable; you could, for example, add a Unify Coroutine Collection to a Unify Coroutine Queue or vice versa.

Unify provides generic and non-generic versions of each type of Unify Collection.

Unify 0.3.0 (Q1/Q2 2021)

Unify Easing Functions

Unify provides many easing functions which come in two forms; you can use the lightweight static easing functions for speed, or if you need something more powerful, Unify also provides a feature rich Coroutine version of each easing function. The Unify Coroutine versions offer all the Unify Coroutine features; delayed start, pause, resume, cancel, restart/reuse, returning a value and looping, plus extra functionality including a progress update event and the choice of an absolute or relative end value. You can even bind a GameObject's transform to an easing function to manipulate a GameObject without any extra code.

As Unify Easing Functions are built on top of Unify Coroutines, they can be added to a Unify Queue or Collection and controlled in the same way as a standard Unify Coroutine.

Unify's easing functions will support easing of the following types; float, Vector2, Vector3, Vector4, Quaternion and Color.
